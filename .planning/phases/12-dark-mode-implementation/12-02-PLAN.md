# Plan 12-02: Apply Dark Mode Classes to All Pages

**Phase:** 12 - Dark Mode Implementation
**Created:** 2026-01-16
**Status:** Not Started

## Context

Plan 12-01 fixed Tailwind v4 configuration by adding the `@variant dark` directive. Now Tailwind generates dark mode utility classes (`dark:bg-*`, `dark:text-*`, etc.). This plan updates all application pages to use those classes.

**Current state:**
- ✅ Tailwind v4 dark mode configured (@variant directive added)
- ✅ Theme toggle verified working (Plan 12-01 test component)
- ✅ CSS variables defined for both themes
- ❌ Pages still use hardcoded light mode colors
- ❌ Components don't respond to theme changes

**User requirement:**
- Dark mode: dark slate blue background (#0f172a / slate-900) with white text
- Light mode: current styling (light gray background #F5F6FA with dark text)
- Match the `/documents` page reference - "absolutely magnificent" dark bluish slate cards
- Must work across all pages: dashboard, inbox, documents, processing, data

**Reference:** `src/app/(main)/documents/page.tsx` - the gold standard for dark mode styling

## Objective

Systematically update all main application pages to use Tailwind dark mode classes, creating consistent theme support across the entire platform.

## Success Criteria

- [ ] All main pages respond to theme toggle
- [ ] Dark mode uses slate-800/900 backgrounds with white text (matches /documents)
- [ ] Light mode unchanged (current styling preserved)
- [ ] Text is readable in both themes (proper contrast)
- [ ] Cards, borders, and backgrounds all theme-aware
- [ ] No hardcoded `bg-white` or `text-dark` without dark: variants

## Pattern Reference

From the `/documents` page that works perfectly:

```tsx
// Page container
<div className="p-6 bg-light-bg dark:bg-slate-900">

  // Cards
  <div className="bg-white dark:bg-slate-800 rounded-design-lg shadow-card border border-gray-200 dark:border-slate-700">

    // Headings
    <h2 className="text-dark dark:text-white">

    // Body text
    <p className="text-text-secondary dark:text-slate-300">

    // Borders
    <div className="border-t border-gray-200 dark:border-slate-700">
  </div>
</div>
```

**Color scheme:**
- Background: `bg-light-bg dark:bg-slate-900` (#F5F6FA → #0f172a)
- Cards: `bg-white dark:bg-slate-800` (#ffffff → #1e293b)
- Borders: `border-gray-200 dark:border-slate-700` (#e5e7eb → #334155)
- Headings: `text-dark dark:text-white` (#1a1a1a → #f8fafc)
- Body text: `text-text-secondary dark:text-slate-300` (#6b7280 → #cbd5e1)

## Implementation Notes

**Class naming pattern:**
```tsx
// Always pair light and dark variants
<div className="bg-white dark:bg-slate-800">        // Backgrounds
<h2 className="text-dark dark:text-white">          // Headings
<p className="text-text-secondary dark:text-slate-300"> // Body text
<div className="border-gray-200 dark:border-slate-700"> // Borders
<button className="hover:bg-gray-50 dark:hover:bg-slate-700"> // Hover states
```

**Colors to use:**
- Never use pure black (#000000) - use slate-900 (#0f172a)
- Cards: slate-800 (#1e293b) not slate-900
- Borders: slate-700 (#334155) for subtle separation
- Text: white (#f8fafc) for headings, slate-300 (#cbd5e1) for body

**What to preserve:**
- Gradient accents (service colors) work in both themes
- Status badges (green/yellow/red) work in both themes
- Icons - add `dark:text-white` or `dark:text-slate-300` as needed
